## 2. 라이브러리 개요

### 주요 기능

* 문법(Grammar) 기반 Parser 생성
* 토큰 분석(Lexical Analysis)
* 구문 분석(Syntax Parsing)
* HQL, SQL, DSL 등 언어 해석 지원

### 사용 목적

ANTLR(ANother Tool for Language Recognition)은 컴파일러, 인터프리터, SQL Parser, DSL(Domain Specific Language) 등의 구현을 위해 사용되는 Parser 생성 라이브러리이다.

ANTLR 2.x 계열은 현재 유지보수 종료(EOL) 상태이며 최신 버전은 ANTLR 4.x 계열이다.

본 라이브러리는 전자정부프레임워크와 직접적인 연관은 없으며 Hibernate 내부 Parser 구현에 사용된다.

### 대표 사용 예시

```java
import antlr.Token;
import antlr.TokenStream;

public class SampleParser {
    // ANTLR 기반 Parser 처리
}
```

실제 프로젝트에서는 직접 사용하기보다 Hibernate 내부 구현에서 사용되는 경우가 대부분이다.

## 3. AS-IS 사용 현황

### 프로젝트 내 사용 여부

* 직접 사용 미확인
* Hibernate 전이 의존성(Transitive Dependency)으로 사용

### 확인 방법

Maven Dependency Tree 확인

```bash
mvn dependency:tree -Dincludes=antlr
```

### 확인 결과

```text
org.hibernate:hibernate-entitymanager:5.6.15.Final
 └─ org.hibernate:hibernate-core:5.6.15.Final
     └─ antlr:antlr:2.7.7
```

### 예상 사용 위치

* Hibernate Framework 내부
* HQL Parser
* ORM Query Parsing

## 6. 버전 업그레이드 검토

### AS-IS 버전

```text
antlr:antlr:2.7.7
```

### TO-BE 버전

```text
antlr:antlr:2.7.7 (유지)
```

### 주요 변경사항

별도 업그레이드 수행 대상 아님.

ANTLR 4.x는 구조가 완전히 변경되었으며 다음과 같은 직접 업그레이드는 불가능하다.

```text
antlr:antlr:2.7.7
→
org.antlr:antlr4-runtime
```

ANTLR 2.x와 ANTLR 4.x는 API 및 Grammar 구조가 서로 호환되지 않는다.

### 소스 수정 여부

* 없음

## 7. 위험요소

### 보안

* 공개된 주요 보안 취약점(CVE) 영향은 확인되지 않음
* Hibernate 내부 사용 라이브러리이므로 직접적인 보안 영향도 낮음

### 운영

* Hibernate 동작에 필수적인 내부 의존성
* 임의 제거 시 HQL 처리 오류 발생 가능

### 호환성

| 항목              | 영향     |
| --------------- | ------ |
| JDK 17          | 영향 없음  |
| Spring 5.3      | 영향 없음  |
| eGovFrame 4.3   | 영향 없음  |
| Hibernate 5.6.x | 필수 의존성 |
